## How to Use This Register

When a breach is investigated and closed, add a row using the following guidance:

| Field | Notes |
|-------|-------|
| Date discovered | Date the breach was first identified (not the date it occurred, if different) |
| Date reported to ICO | Date of ICO notification, or "Not reported — risk assessed as low" |
| Nature of breach | Brief description: e.g. "Unauthorised access to user email addresses via misconfigured API endpoint" |
| Data categories affected | e.g. "Email addresses, organisation names" |
| Number of individuals | Approximate count of data subjects whose data was involved |
| Risk level | Low / Medium / High — per the risk classification in BREACH_RESPONSE.md |
| Actions taken | Summary of containment, remediation, and user notification steps |
| Outcome | ICO outcome if reported; improvements implemented; case closed |

## Retention

This register must be retained for a minimum of **3 years** from the date of the most recent entry. It is confidential and must not be shared externally without legal advice.
