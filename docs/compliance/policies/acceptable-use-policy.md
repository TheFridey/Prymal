# Acceptable Use Policy

## Purpose

Set basic rules for how Prymal systems and admin access are used by founders, staff, and contractors.

## Scope

All people using Prymal's code repositories, production infrastructure, and admin consoles.

## Owner

Founder / Security lead

## Review Frequency

Annually.

## Requirements

- Use named accounts only.
- Do not share credentials or MFA devices.
- Do not store secrets in code, tickets, chat, or screenshots.
- Use company-approved devices and browsers where possible for production administration.
- Report suspected incidents, phishing, or device compromise promptly.
- Production access is for business use only.

## Evidence Expected

- access review records
- incident reports
- acknowledgement record where practical

## Exceptions Process

Any exception must be approved by the founder/security lead and documented with compensating controls.
