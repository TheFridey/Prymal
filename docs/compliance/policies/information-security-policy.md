# Information Security Policy

## Purpose

Set the minimum information security expectations for Prymal's systems, people, suppliers, and operating practices.

## Scope

All Prymal production and staging systems, code repositories, admin consoles, data stores, suppliers, and people with access to them.

## Owner

Founder / Security lead

## Review Frequency

Annually or after a significant incident or platform change.

## Requirements

- Security is built into design, development, deployment, and support activities.
- Production changes must pass technical validation and security preflight checks.
- Access must be least-privilege and reviewed regularly.
- MFA is required for all cloud services that can affect production or customer data.
- Secrets must never be committed to Git or shared in plaintext tickets.
- Incidents, vulnerabilities, backups, and changes must be recorded in the compliance registers.

## Evidence Expected

- evidence register
- access reviews
- vulnerability register
- incident register
- release review records

## Exceptions Process

The founder or delegated security lead may approve a time-bound exception. Record the reason, risk, owner, expiry date, and mitigation in the risk or corrective action register.
